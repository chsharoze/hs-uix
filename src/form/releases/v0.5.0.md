# v0.5.0

Full roadmap release — v0.2.0 through v0.5.0 + backlog, shipped together.

## v0.2.0 — Sections & Layout

### Added
- **Sections** (`sections` prop) — group fields into collapsible `Accordion` containers with `id`, `label`, `fields[]`, `defaultOpen`, and optional `info` tooltip
- **Field groups** (`group` prop on fields) — lightweight non-collapsible grouping with auto-inserted `Divider` + demibold group label between groups
- **Display fields** (`type: "display"`) — render-only fields with no form value, no validation, excluded from submit. Use `render: ({ allValues }) => ...`
- **Form-level read-only** (`readOnly` + `readOnlyMessage` props) — locks all fields, hides buttons, shows warning Alert

## v0.3.0 — Validation & Submit Lifecycle

### Added
- **Async validation** — `validate` can return a `Promise<true | string>`. Field shows loading indicator while in-flight. `validateDebounce` (ms) prevents excessive calls
- **Field-level loading** (`loading` prop on fields) — spinner on individual fields (e.g., while fetching options)
- **Value transform** (`transformValues` prop) — reshape values before submission. Raw values available as `rawValues` in submit helpers
- **Submit lifecycle callbacks** — `onSubmitSuccess(result, { reset, values })`, `onSubmitError(error, { values })`, `resetOnSuccess`
- **Confirmation gate** (`onBeforeSubmit` prop) — intercept submit, return `false` to cancel or `Promise<boolean>` for async confirmation

## v0.4.0 — Field Intelligence

### Added
- **Conditional required** — `required` can be `(values) => boolean`
- **Field-level side effects** (`onFieldChange` on field definitions) — `(value, allValues, { setFieldValue, setFieldError }) => void` for cross-field updates without triggering validation loops
- **Repeater fields** (`type: "repeater"`) — add/remove dynamic rows with sub-fields, `min`/`max` row constraints. Sub-fields support text, select, number, checkbox
- **Custom field types** (`fieldTypes` prop) — plugin registry with `render`, `getEmptyValue`, `isEmpty`. TypeScript `FormBuilderFieldType` accepts custom strings

## v0.5.0 — CRM Integration

### Added
- **`setErrors` ref API** — batch set field errors from server/API responses
- **`useFormPrefill` hook** — maps CRM property values (from `useCrmProperties`) to form initial values via `{ formField: "crmProperty" }` mapping
- **Auto-save** (`autoSave` prop) — `{ debounce?: number, onAutoSave: (values) => void }`. Fires only when dirty

## Backlog

### Added
- **Field debounce** (`debounce` prop on fields) — delays `onChange` callback by specified ms, useful for search-as-you-type
- **CRM data components** — `type: "crmPropertyList"` and `type: "crmAssociationPropertyList"` embed native HubSpot components directly in forms. Hands-off: HubSpot handles editing and auto-saving. No form value or validation
