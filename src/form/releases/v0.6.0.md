# v0.6.0

Audit-driven reliability + API consistency release.

## Breaking Changes
- `dependsOn*` field props were removed. Use `dependsOnConfig`.
- Root repeater props (`addLabel`, `removeLabel`, `renderAdd`, `renderRemove`) were removed. Use `repeaterProps`.
- Form-level legacy button labels (`submitLabel`, `cancelLabel`, `backLabel`, `nextLabel`) were removed. Use `labels`.
- Form-level legacy alert props (`addAlert`, `readOnlyTitle`, `errorTitle`, `successTitle`) were removed. Use `alerts`.

## Fixed
- Correct submit gating for async validation: submit now checks fresh error state after awaiting pending async validators.
- Fixed controlled-mode side-effect updates (`setFieldValue`) so cross-field updates do not clobber the latest changed value.
- Cascading option clearing now propagates transitively across dependency chains (e.g. `A -> B -> C`).
- Auto-save change detection now uses deep comparison instead of reference equality, reducing noisy saves in controlled mode.
- Added cleanup for input debounce timers on unmount.

## Validation Improvements
- Repeater sub-fields are now validated by the form validation engine (submit/step validation).
- Repeater sub-fields now validate on blur/onChange and surface field-level messages.
- Async validation now versions requests and aborts prior validations when supported (`validate(..., { signal })`).
- Async validators now run during submit/step-next validation for untouched visible fields.
- Sync validation no longer eagerly executes `async` custom validators, avoiding duplicate async side effects.
- Added config warnings for common misconfigurations (missing step/layout/section/dependency field names).

## Performance / Rendering Improvements
- Added stable repeater row keys (object identity-based) to reduce row remount churn.
- Cached field lookup map for repeated name-based resolution paths.

## API Additions
- Controlled validation errors: `errors` prop.
- Button label grouping: `labels` prop (`submit/cancel/back/next`).
- Custom action row rendering: `renderButtons(context)`.
- Grouped alert config: `alerts` prop (`addAlert`, `readOnlyTitle`, `errorTitle`, `successTitle`).
- Form wrapper pass-through: `formProps`.
- Field-level disable support: `field.disabled`.
- Dependent config object alias: `dependsOnConfig`.
- Repeater config namespace: `repeaterProps` (including optional row reorder controls).
